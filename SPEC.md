# AI 周报助手 - 产品规格说明书

## 项目概述

AI 周报助手是一个面向个人工程师的 Web 应用，基于 Next.js 构建。能够从 GitHub 和飞书日历中自动采集工作数据，利用智谱 AI 进行智能总结，每周五自动生成结构化的中文周报，并通过飞书机器人推送。同时提供 Web 界面用于查看历史周报、手动触发生成和管理配置。

## 用户故事

### 背景

作为一名工程师，每周需要撰写周报来回顾和总结工作内容。手动整理 commit 记录、PR/Issue 状态和会议信息既耗时又容易遗漏。需要一个自动化工具来完成数据采集和智能归纳，减少重复劳动。

### 核心用户故事

1. **作为**工程师，**我希望**工具自动采集我在 GitHub 上的活动记录，**以便**不需要手动翻阅每个仓库的提交历史。

2. **作为**工程师，**我希望**工具自动获取我本周参加的会议信息，**以便**周报中包含协作和沟通的部分。

3. **作为**工程师，**我希望**AI 能将零散的记录归纳成可读的段落，**以便**周报内容简洁有条理，而非简单的列表罗列。

4. **作为**工程师，**我希望**周报每周五自动生成并推送到飞书，**以便**无需手动触发，节省时间。

5. **作为**工程师，**我希望**周报同时保存到本地文件，**以便**留存历史记录和后续查阅。

## 功能需求

### F1: GitHub 数据采集

- 采集当前用户下所有仓库的活动
- 数据类型：
  - **Commits**：提交信息、仓库名、时间
  - **Pull Requests**：标题、状态（merged/open/closed）、仓库名
  - **Issues**：标题、状态（open/closed）、仓库名
- 按个人活动过滤（仅采集当前用户的操作）
- 时间范围：本周一 00:00 至周五 23:59

### F2: 飞书日历数据采集

- 通过飞书自建应用 API 获取日历事件
- 采集内容：会议标题、开始时间、结束时间
- 仅采集用户参加的会议
- 时间范围：本周一至周五
- **后续扩展**：支持会议纪要/笔记内容

### F3: AI 智能总结

- 使用智谱 AI（GLM 系列）进行内容总结
- 将零散的 commit、PR、Issue 和会议记录归纳成可读段落
- 输出分类结构：
  - **本周完成**：已 merge 的 PR、已关闭的 Issue、关键 commit 总结
  - **进行中**：仍 open 的 PR 和 Issue
  - **会议与协作**：本周参加的会议概要
  - **下周计划**：基于进行中的工作推断（AI 生成建议）
- 输出语言：中文

### F4: 周报生成与格式化

- 输出格式：Markdown
- 包含以下结构：
  ```
  # 周报 - YYYY.MM.DD ~ YYYY.MM.DD
  
  ## 本周完成
  （AI 总结段落）
  
  ## 进行中
  （AI 总结段落）
  
  ## 会议与协作
  （会议列表及概要）
  
  ## 下周计划
  （AI 生成建议）
  ```
- 文件命名：`weekly-report-YYYY-MM-DD.md`

### F5: 飞书消息推送

- 通过飞书自建应用的机器人能力发送消息
- 将周报内容推送给用户
- 支持 Markdown 格式展示（飞书富文本消息）

### F6: 本地文件保存

- 将生成的 Markdown 周报保存到指定目录
- 目录路径可通过配置文件指定
- 按日期归档，便于历史查阅

### F7: 定时任务

- 每周五自动执行周报生成流程
- 执行时间可配置（默认：每周五 18:00）
- 支持通过 Web 界面手动触发生成

### F8: Web 界面

- **仪表盘**：展示最近一期周报概览和生成状态
- **历史周报**：按时间列表查看所有已生成的周报
- **周报详情**：在线阅读单篇周报的完整内容
- **手动触发**：一键生成本周周报
- **设置页面**：管理 GitHub、飞书、智谱 AI 等配置项（后续扩展）

## 非功能需求

- **可配置性**：所有外部服务凭证和参数通过环境变量或配置文件管理
- **错误处理**：单个数据源采集失败不影响其他数据源，周报仍可生成（标注缺失部分）
- **日志**：关键步骤输出日志，便于排查问题
- **幂等性**：同一时间段重复执行不产生重复文件（覆盖或跳过）

## 技术架构

### 技术栈

| 类别 | 选型 |
|------|------|
| 框架 | Next.js 15（App Router） |
| 语言 | TypeScript |
| 样式 | Tailwind CSS 4 |
| 代码规范 | ESLint |
| 运行时 | Node.js |
| 包管理 | pnpm |
| GitHub API | Octokit（`@octokit/rest`） |
| 飞书 API | 飞书开放平台 REST API |
| AI 模型 | 智谱 AI GLM 系列（HTTP API） |
| 定时任务 | Cron Job（Vercel Cron / node-cron 本地部署） |
| 配置管理 | 环境变量（`.env.local`） |

### 配置项

```env
# GitHub
GITHUB_TOKEN=               # GitHub Personal Access Token
GITHUB_USERNAME=            # GitHub 用户名

# 飞书
LARK_APP_ID=                # 飞书自建应用 App ID
LARK_APP_SECRET=            # 飞书自建应用 App Secret
LARK_BOT_WEBHOOK=           # 飞书机器人 Webhook 地址（可选，备用推送方式）
LARK_USER_ID=               # 飞书用户 ID（用于日历查询和消息推送）

# 智谱 AI
ZHIPU_API_KEY=              # 智谱 AI API Key
ZHIPU_MODEL=glm-5           # 使用的模型（默认 glm-5）

# 周报配置
REPORT_OUTPUT_DIR=./reports # 周报保存目录

# 定时任务（Vercel Cron 使用 vercel.json 配置，本地部署使用此变量）
CRON_SCHEDULE=0 18 * * 5   # 定时任务表达式（默认周五 18:00）
CRON_SECRET=                # Cron API 路由鉴权密钥
```

## 智谱 AI 注意事项

- 智谱 AI 推理模型（glm-4-flashx、glm-z1-flash 等）返回内容在 `reasoning_content` 字段而非标准 `content` 字段，需特殊解析处理。
